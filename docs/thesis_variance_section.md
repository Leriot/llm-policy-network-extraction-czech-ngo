## 4.x  Non-Determinism in LLM-Based Coding and Mitigation Strategy

A characteristic feature of transformer-based language models is that their outputs are not guaranteed to be deterministic. Depending on how inference is executed, identical inputs can yield different classifications across repeated calls. This has direct implications for the reproducibility of automated content coding, and requires explicit methodological attention before results can be interpreted with confidence.

To empirically quantify this effect in the present study, a controlled variance test was conducted on a shared sample of rows drawn from the pilot dataset. Two inference environments were compared: a locally hosted instance of Llama 4 Scout running via Ollama on a single machine, and OpenRouter's API-routed access to Mistral-Small. Each model was run repeatedly under two temperature settings — temperature = 0.1 (the setting used throughout the study) and temperature = 0.0, which eliminates sampling randomness by forcing the model to select the single highest-probability token at each step. Runs were conducted with a fixed seed (67) in all conditions.

**Results.** The findings reveal a sharp contrast between the two inference environments. The locally hosted Scout model produced perfectly identical outputs across all three repeated passes, under both temperature conditions, yielding a mean pairwise Cohen's κ of 1.000 and 100% row-level agreement (50/50 rows). Label distributions were entirely stable: 20 collaboration, 22 co-mention, and 8 wrong in every run without exception. The OpenRouter-routed Mistral model, by contrast, showed meaningful run-to-run variation despite identical prompts, seeds, and temperature settings. At temperature = 0.1, mean pairwise κ across four runs was 0.811 (77.5% full agreement), with the *wrong* category showing the widest drift — ranging from 7 to 12 assigned labels across runs. Reducing temperature to 0.0 provided only marginal improvement: κ = 0.799 and 80.0% full agreement, with the *wrong* range tightening to 7–9. Three specific article–NGO pairs (two involving Hnutí Duha, one involving Greenpeace ČR) exhibited instability across all four OpenRouter runs under both temperature conditions, pointing to semantic ambiguity at the excerpt level rather than model randomness. When comparing the two environments directly on matched rows, cross-environment agreement stood at κ = 0.621–0.625 — substantial, but reflecting genuine systematic differences in how the two models interpret borderline cases.

These results demonstrate that the source of observed variance is not temperature-driven sampling stochasticity — temperature zero provided negligible improvement — but rather compute-level non-determinism introduced by API routing: server-side batching, heterogeneous GPU hardware, and floating-point rounding across inference nodes. This is a known and documented limitation of commercial LLM APIs (OpenRouter, 2024) and is distinct from the prompt instability or model quality concerns that temperature is designed to address.

**Implications and ideal practice.** From a strict reproducibility standpoint, the gold standard for this type of research is to run all inference locally on hardware-locked infrastructure — a single physical or virtual machine, a fixed model checkpoint, deterministic CUDA kernels, and a fixed batch size — such that the same input will always produce the same output. The local Scout results demonstrate this is achievable in practice: κ = 1.000 across all repeated passes. For large-scale studies, a dedicated cloud instance with a self-hosted open-weight model (e.g. via vLLM or Ollama on a fixed GPU type) would replicate this property while scaling to corpora far beyond what a consumer machine can process. Researchers using API-routed inference, by contrast, must accept residual non-determinism and build their validation design around it.

**Approach taken in this thesis.** For the present study, OpenRouter was selected as the primary inference backend on practical grounds: the scale of the corpus and the number of models involved — five distinct LLMs were used in combination — would have required prohibitive compute time or cost to run fully locally within the timeframe of a master's thesis. The API-routed approach introduced the variance documented above, but this was controlled through a layered validation design. First, all models were run with a fixed seed and the lowest feasible temperature (0.1) to minimise stochastic drift. Second, disagreements between models were not discarded but flagged and submitted to an LLM judge — a separate inference pass in which a dedicated model adjudicates conflicting labels by reasoning over the excerpt and the codebook. Third, a random stratified sample of judge-resolved cases was subjected to final human validation by the author and a second coder, providing a human-anchored ground truth against which automated labels were calibrated. The inter-rater statistics reported in Section X therefore reflect this full pipeline, not raw single-pass API output. While perfect reproducibility in the computational sense cannot be claimed, the combination of repeated validation passes, LLM adjudication, and human spot-checking constitutes a methodologically defensible approach to the non-determinism inherent in API-based LLM research at master's thesis scale.
